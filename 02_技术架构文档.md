### 2.2 核心接口

```go
// 资源驱动标准接口
type ResourceDriver interface {
    ResourceType() string
    Connect(cfg DriverConfig) error
    HealthCheck() error
    List(ctx context.Context, query Query) ([]Resource, error)
    Get(ctx context.Context, id string) (Resource, error)
    Create(ctx context.Context, spec ResourceSpec) (Resource, error)
    Update(ctx context.Context, id string, spec ResourceSpec) error
    Delete(ctx context.Context, id string) error
    SupportedActions() []string
    ExecuteAction(ctx context.Context, id string, action string, params map[string]interface{}) error
    CollectMetrics(ctx context.Context, id string) ([]Metric, error)
}
```

## 四、Agent 架构

### 4.1 核心职责

```
Agent (每台宿主机部署)
├── 注册：首次启动向平台注册
├── 心跳：每30秒上报资源使用情况
├── 指令执行：接收平台下发的指令并执行
│   ├── libvirt 操作（创建/启停/快照 VM）
│   ├── qemu-img 操作（创建/扩容磁盘）
│   ├── cloud-init ISO 生成
│   ├── iptables 规则管理
│   └── Ceph 命令代理
└── WebSocket：VNC 代理转发
```

### 4.2 通信协议

```
Agent → 平台 (HTTP POST /api/v1/agent/heartbeat)
{
    "agent_id": "uuid",
    "hostname": "ceph-node1",
    "cpu_usage": 45.2,
    "mem_usage": 72.1,
    "disk_usage": 15.3,
    "vms": [{"id": "...", "state": "running"}, ...]
}

平台 → Agent (HTTP POST，仅接收已注册 Agent 的指令)
{
    "action": "create_vm",
    "params": {
        "name": "test-vm-01",
        "vcpus": 2,
        "memory_mb": 4096,
        "disk_size_gb": 40,
        "storage_type": "local",
        "image": "alma9.qcow2",
        "cloud_init": { "hostname": "...", "ssh_key": "..." },
        "network": "br0",
        "ip": "10.0.50.200"
    },
    "signature": "hmac_sha256"
}
```

## 五、数据库核心表

```sql
-- 租户
tenants (id, name, status, quota_cpu, quota_mem, quota_disk, created_at, updated_at)

-- 用户
users (id, tenant_id, username, password_hash, email, status, created_at)

-- 角色
roles (id, tenant_id, name, permissions JSON, created_at)

-- 物理机
hosts (id, name, ip, cpu_total, mem_total, disk_total, status, agent_id, created_at)

-- 虚拟机
virtual_machines (id, tenant_id, host_id, name, vcpus, memory_mb, disk_size_gb,
    storage_type, image_id, network, ip, state, created_at)

-- 镜像
images (id, name, os_type, version, path, size, cloud_init_support, created_at)

-- 快照
snapshots (id, vm_id, name, storage_type, path, size, created_at)

-- 虚拟网络
networks (id, tenant_id, name, type, cidr, gateway, vlan_id, created_at)

-- 安全组
security_groups (id, tenant_id, name, rules JSON, created_at)

-- CLB
load_balancers (id, tenant_id, name, vip, ha_mode, created_at)

-- 浮动IP
floating_ips (id, tenant_id, network_id, ip, vm_id, status, created_at)

-- 审计日志
audit_logs (id, user_id, tenant_id, action, resource_type, resource_id, detail JSON, ip, created_at)
```

## 七、开发边界

### 一期明确不做
- K8s/Docker/Jenkins/GitLab/Nginx 管理（二期）
- 资源计量/计费（二期）
- 敏感操作审批流（二期）
- VXLAN overlay 网络（一期用 VLAN）
- VM 在线迁移（仅支持离线）
- 堡垒机/WebShell 录屏审计（二期）

### 一期必须做
- 本地 qcow2 + Ceph RBD 双存储
- cloud-init NoCloud ISO 初始化
- 存量 VM 只读纳管
- JWT + Redis Token 黑名单
- 6 个预定义角色（含安全管理员）
- Prometheus + Grafana 监控
