## 三、PRD 修正方案

### 3.1 计算模块修正要点

```
修正前（PRD V1.0）                 修正后
─────────────────────────        ─────────────────────────
磁盘：Ceph RBD 云盘               磁盘：本地 qcow2 + Ceph RBD（备份）
创建：选镜像→规格→网络            创建：选镜像→cloud-init配置→规格→磁盘→网络
快照：Ceph RBD 快照               快照：qcow2 snapshot + external snapshot
迁移：RBD 在线迁移                迁移：qcow2 离线迁移 + 增量拷贝
克隆：RBD clone                  克隆：qemu-img create -b backing_file
```

**必须新增的 PRD 需求：**
1. 系统盘创建：从 cloud-init 镜像 qemu-img create -b 创建 backing file
2. cloud-init 配置页：hostname、root密码、SSH公钥、user-data 脚本
3. 数据盘：可选的额外 qcow2 磁盘
4. 磁盘容量监控和告警：监控 `/home/kvm/images/` 使用率
5. 虚拟机磁盘在线扩容：qemu-img resize + guest OS 识别

### 3.2 网络模块修正要点

**必须补充的 PRD 需求：**
1. VLAN 跨宿主机方案：明确物理交换机 VLAN 配置，或采用 VXLAN overlay
2. CLB 高可用：Keepalived + VIP，主备自动切换
3. 浮动 IP 限制说明：NAT/隔离网络仅支持同宿主机绑定；跨宿主机需桥接网络 + GARP

### 3.3 存储模块修正要点

**必须调整的 PRD 需求：**
1. 块存储重新定位：从"Ceph RBD 云盘管理"改为"虚拟机本地磁盘管理"
2. 新增备份管理子模块：对接 kvm-backup pool
3. RGW User 纳管流程：扫描现有用户→标记为存量→保护凭证

### 3.4 权限模块修正要点

1. Token 吊销：引入 Token 黑名单（Redis）+ 定期清理
2. 统一角色定义：5 个预定义角色 + 安全管理员作为跨租户特殊角色

## 四、技术风险矩阵

| 风险 | 概率 | 影响 | 等级 | 缓解措施 |
|------|------|------|------|---------|
| 本地磁盘单点故障导致 VM 不可用 | 中 | 高 | 🔴 | 定期备份到 Ceph kvm-backup pool |
| 混部环境资源争抢（Ceph OSD 与 KVM 争 CPU/内存） | 高 | 中 | 🟡 | 资源基线预留 + cgroup 隔离 |
| qcow2 backing chain 过长导致性能衰减 | 中 | 中 | 🟡 | 定期 blockcommit 合并 |
| 物理交换机 VLAN 配置错误导致网络中断 | 低 | 高 | 🟡 | VLAN 配置变更需审批 + 灰度 |
| iptables 规则膨胀影响网络性能 | 中 | 中 | 🟡 | nftables 替代 + 规则合并优化 |
| Agent 大面积不可达导致管理失效 | 低 | 高 | 🟢 | Agent 心跳监控 + 告警 |
