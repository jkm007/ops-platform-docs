## 一、项目概述

### 1.1 项目定位

企业内部运维内网统一管理平台，对标阿里云/腾讯云控制台体验，实现对内网物理机、KVM 虚拟机、Ceph 集群、K8s、Docker、中间件的统一纳管和自助化交付。

### 1.2 技术栈

| 层级 | 技术 | 版本要求 |
|------|------|---------|
| 后端框架 | Go + Gin | Go 1.22+ |
| 前端框架 | React + Ant Design Pro | Node.js 20+ |
| 数据库 | MySQL | 8.0+ |
| 缓存 | Redis | 7.x |
| Agent | Go 轻量 Agent | Go 1.22+ |
| 监控 | Prometheus + Grafana | 最新稳定版 |
| 代码仓库 | GitLab | http://10.0.50.107 |

## 四、开发规范

### 4.1 后端 Go 规范

| 规范项 | 要求 |
|--------|------|
| Go 版本 | 1.22+ |
| Web 框架 | Gin |
| ORM | GORM |
| 配置管理 | Viper |
| 日志 | zap |
| API 风格 | RESTful，路径 `/api/v1/{resource}` |
| 错误码 | 统一 `{"code": 0, "msg": "success", "data": {}}` |
| 鉴权 | JWT + Redis Token 黑名单 |
| 测试 | 单元测试覆盖率 > 70% |
| 代码规范 | golangci-lint |

### 4.2 前端 React 规范

| 规范项 | 要求 |
|--------|------|
| Node.js | 20.x |
| 框架 | React 18 + TypeScript |
| UI 组件库 | Ant Design Pro |
| 状态管理 | Zustand / React Context |

| 路由 | React Router v6 |
| 样式 | CSS Modules + Ant Design Token |
| 构建工具 | Vite |
| 代码规范 | ESLint + Prettier |

### 4.3 Agent 规范

| 规范项 | 要求 |

### 4.5 数据库规范

| 规范项 | 要求 |

## 五、一期 MVP 开发范围

### 5.1 模块优先级

| 优先级 | 模块 | 功能点 |
|--------|------|--------|
| P0 | **权限与审计** | 登录/用户/角色/多租户 — 所有模块的前置依赖 |
| P0 | **计算模块** | 物理机纳管 + KVM 虚拟机全生命周期（7步向导创建） |
| P0 | **存储模块** | 对象存储 Bucket + Ceph 集群 Dashboard |
| P0 | **网络模块** | 虚拟网络 + 安全组 + CLB + 浮动IP |
| P1 | **Agent** | 宿主机 Agent 注册/心跳/指令执行 |
| P1 | **监控** | Prometheus + Grafana 集成 |

### 5.2 开发顺序建议

```
Phase 1（第1-2周）：权限模块 + 项目框架搭建
  ├── 后端：Gin 项目初始化 + 数据库迁移 + JWT + RBAC
  ├── 前端：Ant Design Pro 初始化 + 登录页 + 布局框架
  └── Agent：项目初始化 + 注册/心跳协议

Phase 2（第3-6周）：计算模块
  ├── 后端：物理机纳管 + VM CRUD + libvirt 驱动 + cloud-init
  ├── 前端：Dashboard + 物理机列表 + VM 7步创建向导 + VNC
  └── Agent：libvirt 执行器 + 资源采集器

Phase 3（第7-9周）：存储模块
  ├── 后端：Ceph RGW Bucket 管理 + Ceph Dashboard + 备份管理
  └── 前端：对象存储页面 + Ceph 管理页面

Phase 4（第10-12周）：网络模块
  ├── 后端：虚拟网络 CRUD + 安全组 + CLB + 浮动IP
  └── 前端：网络/安全组/CLB/浮动IP 页面

Phase 5（第13-14周）：联调 + 测试 + 部署
  ├── 端到端测试
  ├── 在 ceph-node4 部署平台 + Agent
  └── 使用 create-almalinux-vm.sh 模板创建测试 VM
```
